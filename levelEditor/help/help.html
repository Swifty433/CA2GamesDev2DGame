How to use Ruby's Level Editor

Setup

NOTE: The Level Editor can take a long time to load on occasion, if this happens, try unplugging any usb keyboards or input devices

If a file named "level.csv" does not exist yet create it. Copy any platforms from "level.js" (Located in the "game" folder), and move them into the csv, which each platform being on a new line. Below you can see the correct formatting. Incorrect formatting can lead to the program not reading the info correctly!

                    new Platform(0.0, -460.0, 689.0, 367.0, "rgb(255, 0, 255)", "platform"),
                    new Platform(0.0, 0.0, 900.0, 500.0, "rgb(255, 0, 255)", "platform"),
                    new Platform(-5000.0, -500.0, 5000.0, 1020.0, "rgb(255, 0, 255)", "platform"),
                    new Platform(0.0, -500.0, 1316.0, 40.0, "rgb(255, 0, 255)", "platform"),
                    new Platform(1316.0, -792.0, 321.0, 332.0, "rgb(255, 0, 255)", "platform"),
                    new Platform(1200.0, -165.0, 382.0, 254.0, "rgb(64, 0, 64)", "platform"),
                    new Platform(900.0, -300.0, 700.0, 820.0, "rgb(255, 0, 255)", "platform"),
                    new Platform(1631.0, -411.0, 611.0, 459.0, "rgb(242, 30, 46)", "platform"),

Correct formatting means having a comma after each value with a space, before the comman or not having one after it will affect reading the csv.

Your Platform class will need to be altered too. It should have an arguement for tag after color. Your class should look like this:

                    class Platform extends GameObject {
                        constructor(x, y, width, height, color = 'gray', tag = "platform") {
                          super(x, y);
                          
                          this.addComponent(new Renderer(color, width, height));
                          
                          this.addComponent(new Physics({ x: 0, y: 0 }, { x: 0, y: 0 }, { x: 0, y: 0 });
                          
                          this.tag = tag; 
                        }
                      }

With this you should be done and ready to use the editor! :D

Navigation

To navigate around your level, you can use WASD, or the scroll wheel. Holding shift with the scroll wheel will move horizontal, while not holding it will be vertical. Holding left control will increase the speed of scroll, and holding right shift will so it down, so that every scroll is one unit.

Editing Blocks

The current selected block will have a flashing outline around it, and it's info will be shown in the top left. Clicking this will bring you to the block. Clicking and dragging will move that block. Holding control whilst dragging will resize the box based on mouse position. Holding shift will keep it as a square.

Dragging with middle click will change the width of the block, and holding shift will change the height only.

Right clicking and holding will allow you to individually edit attributes of a block. Pressing the key at the top of the box will bring up a pop up that will let you input a value. Pressing cancel will keep the attribute unchanged. This is the only place you can change colour and tag. (Keep in mind that tag will do nothing unless specifically coded for in your game!)

Mistakes can be fixed by pressing control and z.

If a block is hidden behind another, press control and r to flip the drawing order of blocks.

Saving

To save hit control and S, which should show a log onscreen with all the blocks saved. After this happens, open "level.csv" and copy the contents into the platforms array in "level.js".
